Rynk web demo

The page (the example shell) owns the transport: it opens a link (USB Web Serial, or BLE via WebHID over the OS HID link), runs the permanent get_version probe, then dynamically loads the version-matched rynk-wasm package and drives the typed client over a JS byte-link bridge. The session is full duplex: topic pushes (layer changes, key presses, WPM, …) stream into the Topics pane from the moment the session opens, while requests keep running in the Requests pane. WebHID reaches the firmware's RynkHidService (vendor usage page 0xFF60) and rides the existing OS bond, so there is no pairing prompt.
